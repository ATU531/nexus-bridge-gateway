# Nexus Bridge Gateway 项目

## 项目概述

Nexus Bridge Gateway 是一个基于 Spring Boot 3.4.0 和 Java 21 的 Web3 网关服务，作为 Web2 与 Web3 之间的桥梁，提供统一的身份认证、链上数据查询、交易编排和事件监听能力。

## 技术栈

- **Java 版本**: 21
- **Spring Boot**: 3.4.0
- **Web 框架**: Spring WebFlux (响应式编程)
- **网关核心**: Spring Cloud Gateway (核心路由组件)
- **Web3 库**: Web3j 4.12.0
- **并发模型**: WebFlux + 虚拟线程混合模型
- **构建工具**: Maven
- **持久层框架**: MyBatis-Plus (兼顾敏捷开发与极致 SQL 掌控力)
- **数据库**: MySQL(用户与交易记录)
- **缓存层**: Spring Data Redis Reactive (高并发响应式 Nonce 管理)
- **消息队列**: RabbitMQ(事件通知)

## 核心并发与线程模型

NexusBridge Gateway 采用 **『WebFlux + 虚拟线程』** 混合并发模型，在保证极高吞吐量的同时，兼顾阻塞型 I/O 操作的执行效率。

### 入口层 (Ingress)

基于 **Spring WebFlux** 和 **Netty**，利用少量的 EventLoop 线程处理海量的并发连接：

- **非阻塞 I/O**: Netty 的 EventLoop 线程模型确保 I/O 操作不会阻塞
- **高吞吐量**: 单个 EventLoop 线程可处理数千个并发连接
- **响应式编程**: 使用 Mono/Flux 构建全异步的数据流

```java
@RestController
public class QueryController {
    @GetMapping("/balance/{address}")
    public Mono<ApiResponse<BalanceResponse>> getEtherBalance(@PathVariable String address) {
        // 返回 Mono，不阻塞 EventLoop 线程
        return web3QueryService.getEtherBalance(address)
                .map(ApiResponse::success);
    }
}
```

### 业务执行层 (Worker)

**严格规定**: 所有涉及阻塞型 I/O 操作的操作，严禁直接在 Netty EventLoop 线程上执行。

**阻塞型 I/O 操作包括**:
- JDBC 数据库查询 (MyBatis-Plus)
- 链上 RPC 请求 (Web3j)
- 文件系统操作
- 网络 HTTP 调用

### 线程桥接方案

使用 **Java 21 虚拟线程** (`Executors.newVirtualThreadPerTaskExecutor()`) 包裹阻塞操作，最后以 `Mono/Flux` 形式返回给网关层：

```java
@Service
public class Web3QueryService {
    
    public Mono<BalanceResponse> getEtherBalance(String address) {
        return Mono.fromCallable(() -> {
            // 阻塞操作：Web3j RPC 调用
            EthGetBalance ethGetBalance = web3j.ethGetBalance(
                address, DefaultBlockParameterName.LATEST
            ).send();
            
            BigInteger balanceWei = ethGetBalance.getBalance();
            BigDecimal balanceEth = Convert.fromWei(
                new BigDecimal(balanceWei), Convert.Unit.ETHER
            );
            return new BalanceResponse(address, balanceEth.toPlainString(), "ETH");
        })
        .subscribeOn(Schedulers.fromExecutor(
            Executors.newVirtualThreadPerTaskExecutor()
        ));
        // 虚拟线程执行阻塞操作，完成后自动切换回 EventLoop
    }
}
```

### 线程模型优势

| 特性 | 说明 |
|------|------|
| **轻量级并发** | 虚拟线程占用资源极少，可创建百万级线程 |
| **阻塞友好** | 传统阻塞代码可直接在虚拟线程中运行，无需改写为回调式 |
| **自动调度** | 虚拟线程自动在载体线程上调度，无需手动管理 |
| **零成本切换** | 虚拟线程切换成本极低，适合高频 I/O 场景 |
| **兼容现有代码** | 可复用现有的 JDBC、Web3j 等阻塞式库 |

### 数据流示例

```
客户端请求
    ↓
Netty EventLoop (非阻塞接收)
    ↓
Spring WebFlux 路由分发
    ↓
Mono.fromCallable() (切换至虚拟线程)
    ↓
虚拟线程执行 Web3j RPC 调用 (阻塞等待)
    ↓
RPC 响应返回
    ↓
虚拟线程处理结果
    ↓
Mono 自动切换回 EventLoop
    ↓
WebFlux 封装 HTTP 响应
    ↓
Netty 发送响应 (非阻塞)
    ↓
客户端接收
```

### 模块一：核心查询引擎 (Query Engine)

#### 核心逻辑
聚合多链 RPC，提供标准化的只读接口。

#### 解决的痛点
解决不同链 API 不统一、节点不稳定的问题。

#### 1.1 RPC 节点选型

**选型**: Cloudflare Ethereum RPC 节点

**RPC URL**: `https://cloudflare-eth.com`

**选型理由**:
- **高可用性**: Cloudflare 提供全球分布的节点网络，确保服务稳定性
- **低延迟**: 利用 Cloudflare 的边缘计算能力，提供快速响应
- **免费使用**: 无需付费，适合开发和测试环境
- **HTTPS 支持**: 提供安全的加密连接
- **公共节点**: 无需注册认证，简化配置

**备选节点** (生产环境建议):
- Infura: `https://mainnet.infura.io/v3/YOUR_PROJECT_ID`
- Alchemy: `https://eth-mainnet.g.alchemy.com/v2/YOUR_API_KEY`
- QuickNode: `https://YOUR_ENDPOINT.quiknode.pro/YOUR_KEY/`

#### 3.3 Nonce 管理策略

```java
@Service
public class NonceManager {
    
    @Autowired
    private RedisTemplate<String, Long> redisTemplate;
    
    /**
     * 获取下一个可用 Nonce
     */
    public synchronized BigInteger getNextNonce(String address) {
        String key = "nonce:" + address;
        Long cachedNonce = redisTemplate.opsForValue().get(key);
        
        if (cachedNonce == null) {
            // 从链上获取最新 nonce
            BigInteger onChainNonce = web3j.ethGetTransactionCount(
                address, DefaultBlockParameterName.LATEST
            ).send().getTransactionCount();
            
            redisTemplate.opsForValue().set(key, onChainNonce.longValue());
            return onChainNonce;
        }
        
        // 原子递增
        Long nextNonce = redisTemplate.opsForValue().increment(key);
        return BigInteger.valueOf(nextNonce - 1);
    }
}
```

#### 3.4 Gas 策略

| 策略类型 | 说明 |
|---------|------|
| **保守型** | Gas Price = 当前网络均价 × 1.0，适合不紧急的交易 |
| **标准型** | Gas Price = 当前网络均价 × 1.2，平衡速度与成本 |
| **激进型** | Gas Price = 当前网络均价 × 1.5，确保快速打包 |
| **动态调整** | 根据网络拥堵情况自动调整，支持超时重发 |

### 模块六：监控与熔断中心 (Monitor & Circuit Breaker)

#### 核心逻辑
实时监控链上状态和网关健康度，异常时自动熔断保护。

#### 解决的痛点
防止 Gas 价格异常飙升时账户余额被耗尽，提高可观测性。

#### 6.1 监控指标

| 指标类别 | 具体指标 |
|---------|---------|
| **请求指标** | QPS、响应时间、错误率 |
| **链上指标** | Gas Price、节点延迟、区块高度 |
| **业务指标** | 交易量、Gas 消耗、钱包余额 |
| **系统指标** | CPU、内存、连接数 |

#### 6.2 熔断策略

```
┌─────────────────────────────────────────────────────────────┐
│                    熔断器状态机                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│      ┌──────────┐      失败阈值      ┌──────────┐          │
│      │   关闭   │ ─────────────────> │   开启   │          │
│      │ (Closed) │                    │  (Open)  │          │
│      └────┬─────┘                    └────┬─────┘          │
│           │                               │                │
│           │ 成功                            │ 超时          │
│           │                               ▼                │
│           │                          ┌──────────┐          │
│           └───────────────────────── │  半开   │          │
│                                      │(Half-Open)│          │
│                                      └──────────┘          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 6.3 熔断触发条件

- Gas Price 超过设定阈值（如 500 Gwei）
- 连续交易失败次数超过阈值
- 节点 RPC 响应超时

## 技术选型总结

| 组件 | 选型 | 说明 |
|------|------|------|
| **框架** | Spring Boot 3.4 | 主流 Java 微服务框架 |
| **网关** | Spring Cloud Gateway | 核心路由组件，高性能网关 |
| **Web** | Spring WebFlux | 响应式编程，高并发支持 |
| **Web3** | Web3j 4.12 | Java 以太坊开发库 |
| **并发** | WebFlux + Virtual Threads | WebFlux 响应式 + Java 21 虚拟线程混合模型 |
| **持久层** | MyBatis-Plus | 兼顾敏捷开发与极致 SQL 掌控力 |
| **数据库** | MySQL 8.0 | 关系型数据存储 |
| **缓存** | Spring Data Redis Reactive | 高并发响应式 Redis，支持 Nonce/Gas 缓存 |
| **消息队列** | RabbitMQ | 事件通知 |
| **监控** | Prometheus + Grafana | 指标采集与可视化 |
| **熔断** | Resilience4j | 熔断降级框架 |
